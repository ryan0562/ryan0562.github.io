# 解疑

# 为什么需要子模块（large-screen-modularity）

  • 实时查看当前开发的模板在编辑器内的数据与样式
  • 解决编辑器升级后同步问题，防止开发的模板与编辑器异版本（场景：模板在开发过程中，通用代码或者依赖更新了，导致模板报错）
  • 解决调试问题：模板跟编辑器解耦，需要通过上传才能调用，所以每次修改模板代码都需要重新上传一次，子系统能直接调用，不需要每次都上传
  • 解决通用代码库（通用方法、组件、资源等）需要一致等问题

# 模板、模型、工程数据如何交互的

  1. 在编辑器项目public/index.html内添加了全局通用库
<!-- vue -->
  <script src="/static/bigScreen/libs/vue.js"></script>
  <!-- vuex -->
  <script src="/static/bigScreen/libs/vuex.js"></script>
  <!-- vue-router -->
  <script src="/static/bigScreen/libs/vue-router.js"></script>
  <!-- axios -->
  <script src="/static/bigScreen/libs/axios.js"></script>
  <!-- element-ui -->
  <link rel="stylesheet" href="/static/bigScreen/libs/element-ui/index.css">
  <script src="/static/bigScreen/libs/element-ui/index.js"></script>
  <script type="module">
    // Vue.use(ELEMENT, {
    //   size: 'small'
    // });
    Vue.prototype.$ELEMENT = {
      size: 'small'
    };
  1. 在所有项目的vue.config.js内过滤掉通用库，包括（模板、模型、工程）
configureWebpack: {
  ...,
  externals: {
    vue: 'Vue',
    vuex: 'Vuex',
    'element-ui': 'ELEMENT',
    'vue-router': 'VueRouter',
    'js-cookie': 'Cookies',
    axios: 'axios',
    '@form-create/element-ui': 'formCreate'
  }
}
  1. 基于VUEX来管理全局数据
state: {
  // 是否开发模板
  developmentTemplates: true,
  /* 控制器 */
  config: {},
  /* 大屏模型 */
  bigScreenModel: {
    model: [],
    template: {}
  }
}
