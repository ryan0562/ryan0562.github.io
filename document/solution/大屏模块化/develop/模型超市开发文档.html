# 目前组件超市库使用说明

开发注意事项

  1. 开发组件前，需要先确定搭载的模板内保留的组件盒子是多大，确定好后再去app.vue里修改大小，然后再开发组件，防止组件上传后大小不一导致返工。
  2. 模型上传后会按meta.json里的key作为模块唯一标识，相同的key会覆盖之前的模型，不通的key会创建新的模型。

# 如何使用组件超市库：

  • 总体打包 large-screen-modules-Demo运行npm run lib会生成一个lib总文件

在项目index.html中引入modules.umd.js后直接引入使用即可

<!-- index.html -->
<script src="modules.umd.js"></script>

<!-- .vue内 -->
<dxabhd-cghj></dxabhd-cghj>
  • 分模块打包（目前大屏编辑前后端使用方式） 在组件超市库项目运行npm run module-lib,会将pagecks里每个组件文件打包生成lib文件（包括每个模块组件文件和整个模块的FirstLibrary-0.1.5.zip压缩包，压缩包可在大屏编辑器系统模型超市直接上传更新模块）；

# 动态模型（按需加载）

  • 模型使用
<dynamic-module class="component-item"  :entity="metaJson" >
  • 模型代码
<template>
  <div class="module-wrapper" @click="handleClick">
    <component v-if="showModule" :is="entity.key" v-bind="moduleProps" v-on="moduleEmits" class="dynamic-module"/>
    <img v-else-if="showPreview" :src="modulePreviewImg" class="preview-image"/>
  </div>
</template>
<script>
export default {
  name: 'DynamicModule',
  emits: ['moduleClick'],
  data() {
    return {
      loadedModules: [],
      moduleLoaded: false,
    }
  },
  props: {
    entity: {
      type: Object,
      required: true,
      default: () => {}
    },
    showPreview: {
      type: Boolean,
      default: true
    },
    showEmpty: {
      type: Boolean,
      default: false
    }
  },
  computed: {
    showModule() {
      return !this.showEmpty && !this.showPreview && this.moduleLoaded;
},
// 组件props用于模块接口查询
    moduleProps() {
      const { props } = this.entity;
      let pMap = {};
      props?.forEach(p => {
        if (p.type === 'function') {
          pMap[p.key] = ((value) => {
            let { useMock, mock, api } = value;
            if (useMock === undefined || useMock) {
              return () => new Promise(resolve => resolve(mock));
            }

            if (!api) return () => Promise.reject('数据接口未定义');
            return (params) => {
              const { url, method, filter } = api;
              const headers = (filter.headers ? eval(filter.headers) : () => {})();
              const requestFilter = filter.request ? eval(filter.request) : (headers, params) => { return {headers, params} };
              const responseFilter = filter.response ? eval(filter.response) : (res) => res;
              const req = requestFilter(headers, params);
              let p;
              if (method.toLowerCase() === 'get') {
                p = this.$http.get(url, { params: req.params}, req.headers);
              } else {
                p = this.$http.post(url, req.params, req.headers);
              }
              return p.then(res => responseFilter(res));
            }
          })(p.value);
        } else {
          pMap[p.key] = p.value;
        }
      });
      return pMap;
    },
    moduleEmits() {
      const { emits } = this.entity;
      let eMap = {};
      emits?.forEach(e => eMap[e.key] = e.value);
      return eMap;
    },
    modulePreviewImg() {
      const { path, previewImg } = this.entity;
      return `${path}${previewImg}`;
    }
  },
  methods: {
    handleClick() {
      this.$emit('moduleClick', this.entity);
},
// 加载模块打包js文件
    loadModuleJs(entityData) {
      return new Promise(resolve => {
        const { key, path, entry, timestamp } = entityData;
        if (window[key]) {
          const checkInstall = () => {
            if (window[key]['__loading__']) {
              setTimeout(() => checkInstall(), 1);
            } else {
              resolve();
            }
          }
          checkInstall();
        } else {
          window[key] = { 
            __loading__: true
          };
          let entryJs = document.createElement('script');
          entryJs.onload = () => {
            resolve();
          }
          entryJs.src = `${path}${entry}?_t=${timestamp}`;
          document.body.appendChild(entryJs);
        }
      });
    }
  },
  created() {
    if (!this.showPreview) {
      this.loadModuleJs(this.entity)
      .then(() => {
        this.moduleLoaded = true;
        console.log(`load module ${this.entity.key} success.`)
      });
    }
  }
}
</script>
<style lang="less" scoped>
.module-wrapper {
  width: 100%;
  height: 240px;
  display: block;
  margin: 8px;
  .dynamic-module {
    margin: auto;
    overflow: hidden;
  }
  .preview-image {
    margin: auto;
    width: 90%;
    height: 90%;
    object-fit: cover;
  }
}
</style>

注：如果模块静态文件引入就不需要调用loadModuleJs方法动态引入

# 新增模块组件

  1. 如果新增模块组件先在packages新增模块目录主类型文件目录然后在主类型目录下新增组件（组件结构如下图）

  1. 在packages目录下的index.js文件中引用注册组件，meta.json中需要完善配置组件然后将配置内容加入到public目录下的meta.json文件modules里，mock.json组件数据结构，README.md组件使用说明。

# 删除组件

如果删除整个模块直接删除packages下的整个模块目录，如果是单个模块组件就删除packages模块里的组件，然后删除packages目录下的index.js文件组件的引用和注册代码
