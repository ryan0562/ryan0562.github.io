# 介绍

编辑器基于Vue，使用了公司统一基础框架SDF，主要用于构建大屏、数据展示页面，具有多种类型组件可供使用

针对大屏业务提供可视化编辑器，结合模板库、模型超市，形成一套完整的可视化编辑工具，从而快速、高效的编辑出业务需要的大屏工程

完整的工具链目前包括：工程库、模板库、模型超市、大屏编辑器

# 用前必看

使用前应阅读以下提示，这样出现相应问题后可以快速解决。

宽高异常

组件的默认宽高都是100%，可以根据父容器宽高进行自适应，但在某些情况下组件宽高可能表现异常(出现滚动条、偏移等)，这种情况一般是组件没有自适应父级宽高，组件开发者适配自适应即可。

# 特性

模板库、模型超市都有独立的仓库，可以独立开发、打包，并通过上传来接入使用

# 优势

  • 快速、高效搭建大屏工程
  • 统一模板、组件规范，解决不同组件、模板无法复用、使用成本高等问题
  • 统一化、规范化模型、模板
  • 易操作，学习成本非常低，贴合正常操作习惯。看一遍即能上手操作
  • 模板、模型超市解耦，形成单独的项目、按需上传

# 意义

  • 提高售前、产品快速展示大屏给客户观看，从而占据项目承接先机，提高成功率
  • 减少开发人员开发大屏的时间损耗、人员成本
  • 统一模板、组件规范，解决不同组件、模板无法复用、使用成本高等问题
  • 统一、规范的使用方式方法，便于管理，减少项目迁移以及交接的成本
  • 易操作，学习成本非常低，贴合正常操作习惯。看一遍即能上手操作
  • 模板、模型超市都可以通过上传方式载入到项目内，便于在公司内部开发完成后，直接上传到现场，不需要现场重新开发代码以及重新打包

# 常见问题

# 构建包体积庞大

# 图片、字体、视频等通用资源文件是否打包到了组件内

检测方法：查看打包后的dist文件内是否存在非组件自身以外的资源文件。

解决方案：

  1. CDN(推荐)：把所有通用资源放在服务器上，线上通过nginx代理访问。本地开发通过proxy访问。
  2. 模版原型挂载：把所有资源文件引入单个js文件，引入到模板的入口文件。

# 通用库是否打包进了组件内

检测方法：打开打包后的dist/index.umd.min-report.html文件，查看是否有node_modules的通用库

解决方案：

  1. 单个大屏的通用库放入模板库，挂载到模板入口文件
  2. 多个大屏的通用库放入工程库，挂载到工程入口文件

# 通用组件是否打包进了组件内

检测方法：组件代码里检查是否引入了通用组件

解决方案：

  1. 单个大屏的通用组件放入模板库，挂载到模板入口文件
  2. 多个大屏的通用组件放入工程库，挂载到工程入口文件

# 通用样式是否打包进了组件

检测方法：组件代码里检查是否存在样式import

解决方案：样式文件放入模板库，挂载到模板入口文件
