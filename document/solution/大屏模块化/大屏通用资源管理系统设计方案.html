# 大屏通用资源管理系统设计方案

目的：减少对资源的重复引用及加载，统一服务。

# 本期

# 存储系统

  • 使用文件存储服务器来存放各种类型的资源。
  • 目录结构：
    • 组件库：以组件key作为维度，图片、视频、音频等媒体资源放入：media\img | media\mp4?
    • 第三方库：以库名为维度，第二层添加版本号放入：libs

# API服务

  • 提供RESTful API接口，允许用户上传、下载、删除和查询资源。
  • API应支持多格式的媒体和文件类型，并在上传时进行必要的验证和格式检查。
  • 对于图片和媒体资源，可能需要提供额外的处理和转换服务，如生成缩略图、转码等

# 搜索和元数据管理：

  • 允许用户为资源添加和搜索元数据，如标题、描述、标签等。
  • 提供高级搜索功能，如按类型、日期、标签等条件筛选资源

# 下期

# 缓存：

  • 使用内容分发网络（CDN）来加速静态资源（如JS、CSS和图片）的加载速度。
  • 对于频繁访问的资源，可以在API层或CDN上实现缓存策略。

# 访问控制：

  • 实现基于角色的访问控制（RBAC），确保只有授权的用户或应用程序可以访问特定的资源。
  • 可能需要支持私有和公开两种类型的资源，对于公开资源提供可共享的URL。

# 版本控制和回滚：

  • 实现资源的版本控制，允许用户查看和回滚到之前的版本。
  • 对于代码库类型的资源，可能需要集成Git或其他版本控制系统。

# 监控和日志：

  • 实施系统性能和可用性的监控，包括存储使用情况、API响应时间等指标。
  • 记录详细的访问日志和操作日志，以便进行问题排查和审计。

# 垃圾清理

# 引用计数法

设计一个"引用表"，保存所有组件的引用次数。如果一个组件的引用次数是0，就表示这个值不再用到了，如果长时间引用次数为0（半年？），因此可以将这个组件的静态资源删除

# 标记清除法

上传组件资源后给与一个标记，在引用时清理标记，失去引用时添加标记（删除组件、删除工程、删除工程引用等），如果标记存在时长超过一定时间（半年？），就给与资源删除
